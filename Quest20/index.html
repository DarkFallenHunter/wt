List C#

Вступление

Данная статься посвящена знакомству с классом List в C#. Будут рассмотренны некоторые его особености.

Отличия от массива

Массивы прекрасно подходят для хранения фиксированного списка значений или ссылок. Но при необходимости добавления нового элемента или удаления существующего, возникают проблемы, и коллекции позволяют их решить. Новые возможности:

  1. Добавить новый элемент. Метод Add().
  2. Узнать, содержится ли объект определённого типа в коллекции. Метод Contains().
  3. Определить индекс элемента. Метод IndexOf().
  4. Удалить элемент из коллекции. Метод Remove().
  5. Отсортировать коллекцию. Метод Sort().

Сортировка

Обобщённая коллекция List содержит метод Sort(), который позволяет сортировать её содержимое по заданному признаку. Существует два варианта использования этого метода:

  1. Тот класс, из экземпляров которого состоит List должен наследоваться от интерфейса IComparable, который содержит метор CompareTo(). Этот метод сравнивает данный экземпляр класса с другим экземпляром.
    Пример реализации: 

    class Duck : IComparable<Duck>
{
  public int Size;
  public KindOfDuck Kind;

  public int CompareTo(Duck duckToCompare)
  {
    if (this.Size > duckToCompare.Size)
      return 1;
     else
      if (this.Size < duckToCompare.Size)
         return -1;
      else
         return 0;
  }
}

  2. Создать класс, наследуемый от интерфейса IComparer, содержащего метод Compare(), сравнивающий 2 экземпляра заданного класса, и вызвать Sort() с экземпляром этого класса в качестве аргумента.
    Пример реализации: 

    class DuckComparerByKind : IComparer<Duck>
{
  public int Compare(Duck x, Duck y)
  {
    if (x.Kind < y.Kind)
      return -1;
    if (x.Kind > y.Kind)
       return 1;
    return 0;
  }
}

Заключение

Коллекции позволяют решить проблемы, возникающие в массивах из-за их фиксированной длинны. Благодаря такому классу, как List, можно более гибко оперировать данными, не взирая на их количество.
